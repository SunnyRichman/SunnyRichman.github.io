SQL JOIN Operators

SQL JOIN

  • SQL join clause combines columns from one or more tables in a relational database
  • Use when you want to have data from more than one tables
SELECT [columns] FROM tablename t1
[INNER|LEFT OUTER|RIGHT OUTER|FULL OUTER|CROSS]
JOIN table2 t2 ON conditions_of_join
[WHERE [conditions]] [GROUP BY [columns]]
[HAVING [conditions]] [ORDER BY [columns] ASC|DESC]
[LIMIT N];

SQL Set Operators

  • SQL SET operators combines the results of two (SELECT) queries
  • The number of columns needs to match between queries, and the data type of each column needs to be compatible.

Union-compatible

SELECT the_first_select_query
SET_OPERATOR
SELECT another_select_query;

Set Operator Emulations

  • Some DBMS do not support INTERSECT and MINUS operators
  • Methods for Emulations
    • INTERSECT Operator using IN Operator
    • INTERSECT Operator using DISTINCT and INNER JOIN Clause
    • MINUS Operator using NOT IN Operator
    • MINUS Operator using LEFT OUTER JOIN Clause

RECAP: IN Operator

IN: Match multiple values in a WHERE clause (i.e., multiple OR conditions)

Example:

SELECT DISTINCT city, state FROM Customers
WHERE state NOT IN ('PA', 'CA', 'NY') AND country = 'USA';

Match the values of the column with the multiple values

SELECT DISTINCT city, state, country FROM Customers
WHERE (city, state, country) IN(
	SELECT DISTINCT city, state, country FROM Offices
	WHERE state IS NOT NULL);

Match the set of the columns (using parenthesis to group the columns) with
the multiple records from another queries.
