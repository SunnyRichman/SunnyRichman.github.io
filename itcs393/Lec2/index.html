Structured Query Languages

  • SQL is the standard language for manipulate with Relational Databases
  • Five main SQL sublanguage command include

DDL - Data Definition Language

DML - Data Manipulation Language

DQL - Data Query Language

DCL - Data Control Language

TCL - Transaction Control Language

Database-level Operations

CREATE [DATABASE | SCHEMA] [IF NOT EXISTS] db_name [CREATE OPTION];

CREATE OPTION:{[DEFAULT] CHARACTER SET [=] charset_name | [DEFAULT] COLLATE [=] collation_name | ENCRYPTION [=] {'Y' | 'N'} }

Character Set

A set of symbols and encodings. E.g., utf8mb4 means 4-byte UTF-8 Unicode Encoding (support all Asian languages).

Collation

A set of rules for comparing the encoding in the given character sets, i.e., order of symbols may be different in the different collation. E.g.,

  • Case-insensitive (_ci): A = a
  • Case-sensitive (_cs): A ≠ a
  • Accent-insensitive (_ai): a = ấ
  • Accent-sensitive (_as): a ≠ ấ

💡: The default MySQL server character set and collation are utf8mb4 and utf8mb4_0900_ai_ci.

	CREATE DATABASE IF NOT EXISTS itcs393db DEFAULT CHARACTER SET utf8mb4 DEFAULT COLLATE utf8mb4_0900_ai_ci; -- with create option
	CREATE DATABASE IF NOT EXISTS itcs393db; -- w/o create option
USE db_name; -- sets the current working database
SHOW DATABASES; -- lists all available databases in the server

ALTER [DATABASE | SCHEMA] db_name [ALTER OPTION];

ALTER OPTION:{ [DEFAULT] CHARACTER SET [=] charset_name | [DEFAULT] COLLATE [=] collation_name | [DEFAULT] ENCRYPTION [=] {'Y' | 'N'} | READ ONLY [=] {DEFAULT | 0 | 1} };

ALTER DATABASE itcs393db COLLATE utf8_general_ci; -- example

Table-level Operations

CREATE TABLE [IF NOT EXISTS] tbl_name(
	col_name data_type [col_constraint],
	...
	[PRIMARY KEY (column2 [,column2])]
	[REFERENCES tablename] [, CONSTRAINT constraintName]
);

Basic Data Types and Constraints

  • Numeric Data Types: INT | SMALLINT | DECIMAL
  • Date and Time Data Types (Displayed as zeros value):
    • DATE: ‘0000-00-00’
    • TIME: ‘00:00:00’

    • DATETIME: ‘0000-00-00 00:00:00’
    • TIMESTAMP: ‘0000-00-00 00:00:00’
    • YEAR: 0000
  • String Data Types: CHAR | VARCHAR | ENUM | BLOB
  • SQL constraints are used to specify rules for the data in a table
  • Used when create or modify table.
  • SQL Column Constraints
    • NOT NULL - a column cannot store NULL value
    • UNIQUE - each row for a column must have a unique value
    • PRIMARY KEY – a column or more columns have a unique identity and cannot be NULL value. (NOT NULL and UNIQUE constraints)
  • SQL Constraints
    CONSTRAINT fk_PerOrders FOREIGN KEY (PID) REFERENCES Persons (PID);
    CONSTRAINT chk_PersonGender CHECK (Gender in ('M', 'F'))
    USE itcs393db;

-- The sequence of creating tables is important, i.e., create the table containing primary key first.
CREATE TABLE IF NOT EXISTS staff(
	staffID INT,
	nationalID CHAR(13) UNIQUE NOT NULL,
	firstname VARCHAR(50) NOT NULL,
	lastname VARCHAR(50) NOT NULL,
	address VARCHAR(200),
	city VARCHAR(50),
	CONSTRAINT pk_staff PRIMARY KEY (staffID)
);
    CREATE TABLE IF NOT EXISTS department(
	departmentID INT,
	dept_name VARCHAR(50) UNIQUE NOT NULL,
	managerID INT,
	CONSTRAINT pk_dept PRIMARY KEY (departmentID)
	CONSTRAINT fk_manager PRIMARY KEY (managerID) REFERENCES staff(staffID)
);

DROP TABLE [IF EXISTS] tbl_name [, tbl_name] ...;

-- The sequence of creating tables is important, i.e., drop the table containing foreign key first.
-- Example

DROP TABLE IF EXISTS department;
DROP TABLE IF EXISTS staff; -- or

DROP TABLE IF EXISTS department, staff;
-- Changing any structures of a table
ALTER TABLE tbl_name {ADD | MODIFY | CHANGE | DROP} ...;
  • ALTER can be used in
    • Table definitions
    • Table constraints
    • Column definition
    • Column constraints

ALTER Column Definitions - ADD: Add a New Column to an Existing Table

ALTER TABLE tbl_name ADD COLUMN col_name col_def [FIRST | AFTER existing column];

-- Example
ALTER TABLE staff
ADD COLUMN phone VARCHAR(10) AFTER address, -- phone was added as a new column to the table staff and put after address
ADD COLUMN country VARCHAR(10) AFTER city; -- country was added as a new column to the table staff and put after city

ALTER Column Definitions - MODIFY: Modify Existing Columns

ALTER TABLE tbl_name MODIFY col_name new_col_def [FIRST | AFTER existing column];

-- Example
ALTER TABLE staff
MODIFY COLUMN address VARCHAR(150) NOT NULL, -- address was modified into VARCHAR(150) and have constraint NOT NULL
MODIFY COLUMN lastname VARCHAR(10) NOT NULL AFTER firstname; -- lastname was moved after firstname

ALTER Column Definitions - CHANGE: Rename the Existing Columns

ALTER TABLE tbl_name CHANGE COLUMN old_col_name new_col_name col_def [FIRST | AFTER existing column];

-- Example
ALTER TABLE staff
CHANGE COLUMN firstname staff_fname VARCHAR(50) NOT NULL, -- firstname was renamed as staff_fname
CHANGE COLUMN lastname staff_lname VARCHAR(50) NOT NULL; -- lastname was renamed as staff_lname

ALTER Column Definitions - DROP: Drop the Existing Columns

ALTER TABLE tbl_name DROP COLUMN column_name;

-- Example
ALTER TABLE staff DROP COLUMN country, -- country was removed from the table staff

ALTER Constraints - DEFAULT: Set A Default Valur for the Table

ALTER TABLE tbl_name ALTER column_name [SET DEFAULT default value | DROP DEFAULT];

-- Example
ALTER TABLE staff ALTER city SET DEFAULT 'BKK', -- city has the default value, filled in when there is no value input when inserting data records

ALTER Constraints - CHECK: Create a Condition or expression to check for the column

ALTER TABLE tbl_name [ADD CONSTRAINT chk_const_name CHECK (expression)| DROP CONSTRAINT chk_const_name];

-- Example
ALTER TABLE department ADD CONSTRAINT chk_deptID CHECK (departmentID >= 1); -- Table department has constraint chk_deptID to check if department ID is greater than or equal to 1

ALTER Constraints - PRIMARY KEY: Create a Constraint for Primary Key

ALTER TABLE tbl_name [ADD PRIMARY KEY (column_name)| DROP PRIMARY KEY];

-- Example
ALTER TABLE staff ADD PRIMARY KEY (staffID); -- staffID can be set to a primary key of the table

ALTER Constraints - FOREIGN KEY: Create a Foreign Key and Constraint

ALTER TABLE tbl_name ADD CONSTRAINT FOREIGN KEY (column_name) REFERENCES ref_tbl(ref_col_name;

-- Example
ALTER TABLE staff ADD COLUMN s_deptID INT; -- s_deptID was added to the table staff, then
ALTER TABLE staff ADD CONSTRAINT fk_staffdept FOREIGN KEY (s_deptID) REFERENCES Department (departmentID); -- the foreign key was added to refer to departmentID in the department table
  • Referential action can be additional added for FOREIGN KEY during CREATE and ALTER statement
ActionsDescription
CASCADEDelete or update the row from the parent table and automatically delete or update the matching rows in the child table.
SET NULLDelete or update the row from the parent table and set the foreign key column or columns in the child table to NULL.
RESTRICT | NO ACTIONRejects the delete or update operation for the parent table (Specifying RESTRICT or NO ACTION is set to the default when not specifying referential action.)

ALTER TABLE staff DROP CONSTRAINT fk_staffdept;

ALTER TABLE staff ADD CONSTRAINT fk_staffdept FOREIGN KEY (s_deptID) REFERENCES Department(departmentID)
	ON DELETE CASCADE
	ON UPDATE CASCADE
	
-- fk_staffdept constraint was removed first, then, add a new constraint was added with referential actions: CASCADE
