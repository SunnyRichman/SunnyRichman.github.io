Data Types

Definition of Data Types

  • A data type is a classification that determines the type of value a variable or column can hold.
  • Data types help ensure data integrity and improve query performance.
  • In databases, data types are essential for accurately storing and processing data.
  • MySQL has a wide range of data types.

    Numeric

    used for storing numbers.

    • INTEGER/INT(size)
    • TINYINT(size)
    • SMALLINT(size)
    • MEDIUMINT(size)
    • BIGINT(size)
    • FLOAT(size, d)
    • DOUBLE(size, d)
    • DECIMAL(size, d)/DEC(size, d)

Exact Numeric (fixed precision)

a data type for storing exact values and performing precise and safe calculations. It is commonly used in monetary or financial systems such as prices, salary, account balances and so on. In MySQL, The DECIMAL (also NUMERIC) types is used to represent exact numeric.

  • decimal, numeric
    • decimal(p[,s])] and numeric[(p[,s])]
    • decimal and numeric are functionally equivalent
  • p (precision): maximum total number of digits to be stored up to
    • Range from 1 to 65.
  • s (scale): the number of decimal digits stored to the right of the decimal point.
    • Range from 0 to 30. In MySQL, s must less than or equal to p.

String

Fixed-Length String: CHAR(n)

  • Fixed-length string with a maximum length of n characters (value 0 to 255)
  • Require exactly n bytes of storage.
    • If the string value is shorter than n characters, the remaining bytes will be filled with blank spaces.
    • If the string value is longer than n characters, mysql will not allow to insert data. (unless the restrict mode is disabled)

Variable-Length String: VARCHAR(n), TEXT

  • Variable-length string with a maximum length of n characters (value 0 to 65,535)
  • It will use storage as necessary to store the actual string value PLUS 1 byte (overhead) storing an actual size value of the string.

How different between VARCHAR(n) and TEXT

  • Key differences between VARCHAR and TEXT.
  • TEXT forces the maximum value to be 65,535 bytes.
  • VARCHAR is based on the actual length value which uses less storage for the shorter string.
  • TEXT is suitable for storing large String values that exceed the maximum length limit of VARCHAR.
  • For larger String, MEDIUMTEXT and LONGTEXT are considered.

Miscellaneous Data Types

Data TypeRangeStorage
BOOLEANTrue/False or 1/01 byte
ENUMA list of up to 65,535 distinct values1 or 2 bytes, depending on the number of distinct values
SETA list of up to 64 distinct values1, 2, 3, 4, or 8 bytes, depending on the number of distinct values
  • ENUM - is used to define a column that can contain a finite set of values.
  • Use to store value, which are defined when the table is created.
  • It is useful when you want to restrict the possible values

Note that when inserting data into an ENUM column, the value must be one of the predefined values. If you try to insert a value that is not in the list, MySQL will throw an error.

  • SET - is used to define a column that can contain a set of values.
  • Use to store zero or more values from a predefined set.
