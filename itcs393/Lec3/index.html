DML: Data Manipulation Language

Commands used to manipulate data in a relational database.

INSERT: used to create a record.

UPDATE: used to change records.

DELETE: used to delete records.

INSERT

INSERT a new tuple

INSERT INTO table(col1, col2, ..., coln)
VALUE (val1, val2, ..., valn);

INSERT INTO department -- w/o specific columns
VALUE (1, "IT", NULL);

INSERT INTO department (departmentID, dept_name) -- with specific columns
VALUE (2, "Academic Affairs") -- another columns go NULL

INSERT a batch of tuples into a single table

INSERT INTO staff
VALUE (1, "1111111111111", "Harry", "Potter", "4 Private Dr.", NULL, "Surrey"),
	(2, "2222222222222", "Dobby", "Doe", "5 Public Dr.", "0022221222", DEFAULT),
	(3, "3333333333333", "Genie", "Kales", "6 Static Dr.", DEFAULT, NULL);
  • DEFAULT will insert the default value of the column that was set during the CREATE table.
  • NULL can be interpreted as unknown or no data.

Violation in INSERT 

INSERT INTO department(departmentID, dept_name)
VALUES (2, "Some Department"); -- Assume that 2 was inserted as departmentID into department and departmentID is the primary key.
-- Error Code: 1062. Duplicate entry ‘2’ for key ‘department.PRIMARY'
INSERT INTO department(departmentID)
VALUES (3); -- Assume that dept_name is was set as NOT NULL
-- Error Code: 1364. Field 'dept_name' doesn't have a default value
INSERT INTO staff
VALUES (4, "1111111111111", "News", "Users", "Somewhere", DEFAULT, DEFAULT);
-- Error Code: 1062. Duplicate entry '1111111111111' for key 'staff.nationalID'
INSERT INTO department
VALUES (3, "Data Center", 5);
-- Error Code: 1452. Cannot add or update a child row: a foreign key constraint fails (`itcs393db`.`department`, CONSTRAINT `fk_manager` FOREIGN KEY (`managerID`) REFERENCES `staff` (`staffID`))

UPDATE

UPDATE an existing tuple

UPDATE tablename SET col_name=expression[, col_name=expression] [WHERE conditionlist ];

UPDATE staff SET staff_fname="H." WHERE staffID=1; -- Use the primary key as a condition to uniquely identify the row to be updated
UPDATE staff SET phone="001-123-4567" WHERE staff_fname="H." AND staff_lname="Potter";
-- Error Code: 1175. You are using safe update mode and you tried to update a table without a WHERE that uses a KEY column...

💡: Safe update mode in MySQL is designed to prevent accidental updates or deletions of data, which require the WHERE clause to use the key attribute to identify the record.

SET SQL_SAFE_UPDATES = 0; -- Turn off
SET SQL_SAFE_UPDATES = 1; -- Turn on

DELETE

DELETE an existing tuples

DELETE FROM staff WHERE staffID = 1; -- Use the primary key as a condition to uniquely identify the row to be deleted
DELETE FROM staff WHERE phone IS NULL; -- If a non-key attribute is used in the condition, the statement works only if the safe update mode is off.

RECAP: Referential Action

Referential action can be added for FOREIGN KEY during CREATE and ALTER statement

Referential ActionsDescription
CASCADEDelete or update the row from the parent table and automatically delete or update the matching rows in the child table.
SET NULLDelete or update the row from the parent table and set the foreign key column or columns in the child table to NULL.
RESTRICT | NO ACTION- Rejects the delete or update operation for the parent table.
- Specifying
RESTRICT (or NO ACTION) is set to the default when not specifying referential action.

Example: RESTRICT

Example: Referential Option are ON UPDATE RESTRICT and ON DELETE RESTRICT

Given code

UPDATE staff
SET staffID=111 -- new staffID
WHERE staffID=1;

-- Error Code: 1451. Cannot delete or update a parent row: a foreign key constraint fails (`itcs393db`.`department`, CONSTRAINT `fk_manager` FOREIGN KEY (`managerID`) REFERENCES `staff` (`staffID`) ON DELETE RESTRICT ON UPDATE RESTRICT)

The update or delete operation are PROHIBITED if there is the references from the child table

Example: CASCADE

Example: Referential Option are ON UPDATE CASCADE and ON DELETE CASCADE

UPDATE staff
SET staffID=111 -- new staffID
WHERE staffID=1;

-- 1 row(s) affected Rows matched: 1 Changed: 1 Warnings: 0
DELETE FROM staff
WHERE staffID=111;

-- 1 row(s) affected

The child table is automatically updated to the new value /deleted accordingly

Example: SET NULL

Example: Referential Option are ON UPDATE SET NULL and ON DELETE SET NULL

UPDATE staff
SET staffID=111 -- new staffID
WHERE staffID=1;

-- 1 row(s) affected Rows matched: 1 Changed: 1 Warnings: 0
DELETE FROM staff
WHERE staffID=111;

-- 1 row(s) affected

The child table is automatically updated /deleted to NULL accordingly

Data Import, Export, and Backup

Table Data Import

  • MySQL supports table data import using CSV and JSON files ⚠️: Database must be created prior to data import
  • CSV stands for “Comma-separated values”, i.e., each column value is separated by a comma (or other separators such as semicolons or tabs)

gender.csv

id,gender
1,Male
2,Female
3,N/A
  • JSON stands for “JavaScript Object Notation”, i.e., a plain text
    written in JavaScript object notation

gender.json

[{"id":4, "gender":"Type4"},
{"id":5, "gender":"Type5"},
{"id":6, "gender":"Type6"}]

Import Data using GUI

  • Import the data from a file to a new table
  • Import the data from a file to an existing table

Import Data into a New Table

  1. Open the Table Import Wizard by right-clicking on Tables.
  1. Browse the Data Source: For example, select the CSV file you wish to import.
  1. Choose the Database and Specify Table Name:
    • Select “create new table”.
    • Provide a name for the new table.
  1. Select Data Encoding: Ensure the encoding matches the data source (e.g., utf8 that supports English, Thai, etc.).
  1. Review the Source Columns and Data Types: Confirm the columns and their data types align with the desired schema.
  1. Proceed and Complete the Import.

Import Data into an Existing Table

  1. Open the Table Import Wizard.
  1. Browse the Data Source: For example, choose a JSON file containing the data.
  1. Select the Existing Table:
    • Choose the table where the data will be imported.
  1. Review Source Columns and Data Types: Ensure compatibility with the existing table schema.
  1. Consider Relationships:
    • Check foreign key (FK) and primary key (PK) constraints.
    • Verify data sequence if dependent on relationships.
  1. Proceed and Complete the Import.

Table Data Export

Using the Table Data Export Wizard

  1. Open the Table Data Export Wizard.
  1. Select the Table Source: Choose the table you want to export data from.
  1. Specify File Path and Type:
    • Define the destination path for the exported file.
    • Select the file type (e.g., CSV).
    • Configure file settings:
      • Field Separator (e.g., , for CSV).
      • Line Separator (e.g., LF(Line Feed), CR(Carriage Return), or CR LF).

Using Administration – Data Export

  1. Open the Data Export Option in Administration.
  1. Select the Database and Tables: Choose the database and specific tables for export.
  1. Choose the Export Type:
    • Dump Data Only: Export data without schema definitions.
    • Dump Project Folder: Exports each table into a separate file, allowing selective restoration but slower performance.
    • Self-contained File: Exports all selected database objects into a single file for convenience.

Export SQL Query Results

  1. Run the SQL Query: Execute the desired query in the SQL editor.
  1. Export the Results:
    • After executing, export the result set to a file.
    • Choose a file type (e.g., CSV, JSON).
    • Specify the destination path and file name.

Backup and Restore file

Backup a Database

  1. Open Administration – Data Export.
  1. Select the Backup Type:
    • Choose "Dump Structure and Data" to include both the database schema and its data.
  1. Export to a Self-contained File:
    • Specify the destination folder.
    • Save the database into a single, self-contained file for easier management.

Restore a Database

  1. Open Administration – Data Import/Restore.
  1. Choose the Restore Target:
    • Select an existing database to restore into or create a new database.
  1. Import the Backup File:
    • Browse and select the previously exported self-contained file.
    • Start the restoration process.
