Basic Data Query Language

SQL Syntax

SELECT [DISTINCT] [columns] [AS alias] FROM tablename
[INNER|LEFT|RIGHT JOIN tablename2 ON conditions]
[WHERE conditions] [GROUP BY columns] [HAVING conditions]
[ORDER BY columns ASC|DESC]
[LIMIT N];

SELECT: *

Use * to retrieve all columns/attributes from a table

Example: Show all offices in the classicmodels database

SELECT * FROM Offices;

SELECT: [col_name]

Use column_name to retrieve the specific columns/attributes from a table

Example: Show all office code, their city, and phone number

SELECT officeCode, city, phone
FROM Offices;

SELECT: alias

Alias are used to give a table, or a column in a table, a temporary name.

Example: Show all office code, their city, and phone number

SELECT officeCode AS "Office Code", city AS "City", phone AS "Contact"
FROM Offices;

SELECT: DISTINCT

Use DISTINCT to show only non-duplicate values of the column

Example: List all job title of the employees

SELECT jobTitle
FROM Employees;
SELECT DISTINCT(jobTitle)
FROM Employees;

SELECT: LIMIT

Use LIMIT to select the TOP N rows from the result. By default, MySQL show only the top 1,000 rows of the result.

Example: Show three offices

SELECT * FROM Offices
LIMIT 3;

SELECT: Computed Column

  • A computed column (or Derived Attribute) is a field that is the result of calculating using one or more existing fields.
  • Use arithmetic operations between columns and generate the computed column

Example: Calculate the total price of each order details

SELECT (quantityOrdered * priceEach) AS totalprice FROM Orderdetails;

SQL Functions

  • Built-in functions applied on string, numeric, date value, i.e., attributes in the table
  • Function may appear anywhere in an SQL statement where a value or an attribute can be used
  • Usually depends on DBMS
  • MySQL (built-in) functions references: https://www.w3schools.com/sql/sql_ref_mysql.asp

Example: SQL Functions

SELECT officeCode, CONCAT(city, ', ', state, ', ', country) AS Location FROM Offices;
SELECT orderDate, YEAR(orderDate), MONTH(orderDate), DAY(orderDate), DAYNAME(orderDate) FROM Orders
LIMIT 5;

SELECT with conditions

  • Use conditions to filter some records with WHERE clause
  • Use SQL operators and functions in WHERE clause to create conditions.
SELECT [columns] FROM tablename WHERE [conditions]
[ORDER BY [columns] ASC|DESC]
[LIMIT N];

SQL Operators: Comparisons

  • Comparison operators: =, <>, >, <, >=, <=
  • Example
SELECT customerName, state, creditLimit 
FROM Customers
WHERE (creditLimit >= 30000 AND creditLimit < 50000)
AND state = "MA";

SQL Operators: IS NULL/IS NOT NULL

• Use IS NULL to check whether the attribute value is NULL

SELECT customerName, state, city FROM Customers
WHERE state IS NULL AND city IS NOT NULL AND country = "Germany";

SQL Operators: IN

  • Checks whether attribute value matches any value within a given list
SELECT * FROM Employees WHERE officeCode IN (1, 2, 3);

SQL Operators: LIKE

  • Used in conjunction with wildcards to find patterns within string attributes
    • Percent (%): any characters
    • Underscore (_): one character
SELECT employeeNumber, lastName, firstName FROM Employees
WHERE (lastName LIKE "P%" OR firstName LIKE "A%") AND employeeNumber LIKE "10__";

SQL Operators: Variable Declaration

  • Use SET and @ to create a local variable, which can be used in the SQL SELECT statement.
SET @mycity = "Tokyo";
SELECT * FROM Offices WHERE city=@mycity;

SELECT: Sorting

  • Use ORDER BY to sort the results by the sort key
SELECT employeeNumber, lastName, firstName FROM Employees
ORDER BY lastname ASC, firstname DESC;
  • Sorting records can be performed more than one field:
    • The first sort field is called the major sort key.
    • The second sort field is called the minor sort key.
